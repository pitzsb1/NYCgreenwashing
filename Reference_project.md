# SEA building Energy and GHG prediction

### 머신러닝 참고 프로젝트 제안서


Info. 

미국 시애틀 시는 2050년까지 '탄소 중립 도시'라는 목표를 달성하기 위해, 비주거용 건물의 탄소 배출량에 깊은 관심을 기울이고 있음. 이에 따라 2015년과 2016년에 걸쳐 정밀한 전수 조사가 수행됨. 하지만 이러한 데이터는 수집하는 데 상당한 비용과 노력이 소요되므로, 개발자는 이미 확보된 데이터를 활용하여 아직 측정되지 않은 건물의 이산화탄소 배출량과 총 에너지 소비량을 예측하는 모델을 연구.

GOAL: 시애틀 건물 데이터로 총 에너지 소비 및 CO₂ 배출량 예측

DATA: 2015~2016년 미국 시애틀 시의 건물 에너지 사용 및 탄소 배출 데이터 
(Kaggle 'Seattle-energy-consumption' 데이터셋)

### 참고 프로젝트 선정 이유

본 프로젝트에서는 건물 특성(예: 층수, 총 면적, 창문 수 등)을 활용하여 건물의 에너지 소비량 및 CO₂ 배출량을 예측하고자 함. 건물은 전체 에너지 사용과 온실가스 배출에서 상당한 비중을 차지하며, 초기 설계 단계에서 이러한 예측은 에너지 효율 개선 및 탄소 배출 저감 전략 수립에 유용함. 추후 진행하려는 팀 프로젝트가 될 "건물 설계 효율 예측 모델(가제)"에 도움이 될 것으로 판단하여 이 프로젝트를 참고하기로 결정.

## 원본 데이터 | Seattle-energy-consumption

2015년 데이터: 47개의 열(변수)과 약 3,340개의 행(건물)

2016년 데이터: 46개의 열과 약 3,376개의 열

개발자는 두 연도의 데이터를 통합하여 분석을 진행했으며, 타겟 변수가 없는 행 등을 제외한 최종 분석 대상은 약 6,700여 개의 데이터를 사용함.

### 데이터 구조와 타겟 변수

독립 변수(Features): 건물 유형(PrimaryPropertyType), 층수, 완공 연도(YearBuilt), 총 면적(PropertyGFATotal), 주차장 면적 등

종속 변수(Targets): 이 프로젝트는 두 가지를 예측하는 멀티 타겟 연구임.

- SiteEnergyUse(kBtu): 건물의 총 에너지 소비량

- TotalGHGEmissions(Metric Tons CO2e): 총 온실가스 배출량

### 결측치 처리

열 삭제: 결측치 비율이 50% 이상인 변수(예: Outlier, YearsENERGYSTARCertified)는 정보력이 부족하다고 판단하여 제거

행 삭제: 타겟 변수가 누락된 데이터는 학습이 불가능하므로 삭제

## 데이터 전처리 및 피처 엔지니어링

1. 필터링 및 이상치 제거

비주거용 건물에 집중하기 위해 주거용 데이터는 제외

에너지 소비량이 0이거나 음수(비논리적) 데이터 필터링

2. 피처 엔지니어링

BuildingAge: YearBuilt(완공 연도)를 현재 시점 기준으로 계산하여 건물의 노후도를 나타내는 변수로 변환

로그 변환: 타겟 변수와 면적 관련 변수들이 심하게 왜곡된 분포를 보였기 때문에, np.log1p를 사용하여 정규 분포에 가깝게 변환 (이는 회귀 모델의 성능을 크게 향상시킴)

3. 인코딩 및 스케일링

One-Hot Encoding: 'Property Type'이나 'Neighborhood' 같은 범주형 변수를 수치화

Standard Scaling: 서로 다른 단위(면적, 연도 등)를 가진 수치형 변수들을 평균 0, 표준편차 1이 되도록 표준화하여 모델이 특정 변수에 편향되지 않도록 방지

### Train / Test Set 분할

분할 비율: 일반적으로 전체 데이터의 80%를 Train set, 20%를 Test set으로 분할

검증 방식: Train set 내에서 5-Fold Cross Validation(교차 검증)을 수행하여 모델의 과적합을 방지하고 일반화 성능을 확보

### **에너지 스타 점수(ENERGY STAR Score)의 처리**
이 프로젝트의 흥미로운 점은 에너지 스타 점수를 포함했을 때와 제외했을 때의 성능을 비교 분석했음. 이 점수는 그 자체로 에너지 효율을 나타내는 지표지만, 실제 서비스 환경에서는 얻기 어려울 수 있다는 점을 고려하여 두 가지 시나리오를 모두 검증함.

## 모델링 분석

### 사용된 주요 알고리즘
개발자는 단일 모델을 사용하지 않고 성능 비교를 위해 점진적으로 복잡한 모델을 적용했음.

- Linear Regression: 기본적인 관계 파악

- Ridge & Lasso: 변수가 많을 때 발생하는 과적합(Overfitting)을 방지하기 위한 규제

- Random Forest Regressor: 여러 개의 결정 트리 평균 결정

- XGBoost (Extreme Gradient Boosting): 실패한 모델을 계속 보완해서 최종 모델 설계

- SVR (Support Vector Regression): 고차원 공간에서 회귀선 분석

## 알고리즘 선정 이유

1. 데이터의 특성
   
딥러닝은 이미지, 영상, 음성 같은 비정형 데이터에서 강력함. 하지만 현재 데이터처럼 숫자와 카테고리로 이루어진 천 개 이상의 행 정도의 규모에서는 트리 기반 모델(XGBoost, Random Forest)이 훨씬 빠르고 정확함. (과적합 역시 방지)
   
2. 비선형 관계 파악
   
건물 면적과 에너지 사용량의 관계는 단순히 직선 형태가 아닐 확률이 높음. (선형 회귀분석을 사용했음에도 불구하고) Random Forest와 XGBoost는 변수 간의 복잡한 상호작용을 자동으로 학습하는 능력이 좋음.

3. 변수 중요도 확인

이 프로젝트의 목적은 단순히 예측하는 것을 넘어 "어떤 건축 요소가 탄소 배출에 큰 영향을 주는가?" 를 알아내는 것임. 트리 기반 모델은 학습 후 어떤 변수가 예측에 가장 기여했는지 순위를 매김. (예: PropertyGFATotal이 가장 중요하다는 결과 등) 이는 모델이 결과를 해석하는 데에 매우 뛰어난 걸 보여 줌. 

4. 이상치 및 데이터 스케일에 대한 강인함

트리 기반 모델은 데이터의 스케일(단위)에 민감하지 않고, 이상치의 영향을 비교적 적게 받음. 시애틀 건물 데이터처럼 건물마다 크기나 용도가 제각각인 경우에 매우 안정적인 결과를 나타냄.

## 성능 평가 분석

### 사용된 성능 평가 지표
개발자는 모델의 예측값이 실제값과 얼마나 차이 나는지 측정하기 위해 다음 세 가지 지표를 핵심적으로 사용함.

- R^2 Score (결정계수): 모델이 데이터의 분산을 얼마나 잘 설명하는지를 나타냄. 1에 가까울수록 모델이 데이터를 완벽하게 설명한다는 뜻이며, 보통 0.7 이상이면 우수한 모델로 평가

- RMSE (Root Mean Squared Error, 평균 제곱근 오차): 예측값과 실제값 차이의 제곱 평균에 루트를 씌운 값. 에러에 큰 가중치를 두기 때문에, 큰 오차를 잡아내는 데 민감함. 값이 작을수록 정확도가 높음.

- MAE (Mean Absolute Error, 평균 절대 오차): 오차의 절대값 평균. RMSE보다 이상치에 덜 민감하여, 모델의 일반적인 오차 수준을 직관적으로 보여 줌.

### 한계 극복을 위한 시도 및 최적화 전략
단순히 모델을 돌리는 데 그치지 않고, 성능을 극한으로 끌어올리기 위해 개발자는 다음과 같은 정교한 기법들을 도입함.

1. 로그 변환의 재발견 

데이터 분석 초기, 에너지 사용량과 탄소 배출량 데이터가 특정 범위에 몰려 있는 왜곡된 분포를 보였음. 이를 그대로 학습하면 오차가 커지는 한계가 있어, 모든 타겟 변수에 로그를 취해 학습시키고 결과를 다시 역변환하는 방식을 선택. 이 과정이 RMSE를 낮추는 데 가장 결정적인 역할을 함.

2. 하이퍼파라미터 튜닝: GridSearchCV모델의 '설정값'을 최적화하기 위해 GridSearchCV를 사용했습니다.

_GridSearchCV의 작동 원리_

Grid(격자) + Search(찾기) + CV(교차 검증). 이름 그대로 모든 경우의 수를 격자 모양으로 짜놓고 하나씩 다 대입해 보는 방식.

XGBoost/Random Forest: 트리의 개수(n_estimators), 최대 깊이(max_depth), 학습률(learning_rate) 등을 대입하여 최적의 조합을 발견. 

효과: 기본 설정(Default) 모델보다 테스트 데이터에 대한 일반화 성능이 약 5~10% 향상됨.

3. 교차 검증

데이터 수가 1,575개로 아주 많지는 않기 때문에, 특정 데이터셋에만 잘 맞는 모델이 나오는 것을 방지하고자 5-Fold Cross-Validation을 수행, 즉 데이터를 5개 그룹으로 나눠 번갈아 가며 검증함으로써 모델의 신뢰성을 확보함.

4. 변수 선택

모든 변수를 다 넣는 것이 최선이 아님을 인지하고, 모델 기여도가 낮은 변수들을 쳐내는 과정을 반복. 특히 '에너지 스타 점수'의 유무가 모델 성능에 미치는 영향을 분리해서 측정함으로써, 데이터 수급이 어려운 실무 환경까지 고려한 분석까지 진행.

### 최종 성능 요약 (예상치 기준)

분석 결과 XGBoost 모델이 R^2 기준 0.8 내외의 높은 성적을 거두었으며, 단순 선형 회귀보다 월등한 성능을 보임. 결론적으로 개발자는 단순히 알고리즘을 복잡하게 가져가는 것보다 데이터의 분포를 바로잡고(로그 변환), 파라미터를 정밀하게 튜닝하는 것이 성능 향상의 핵심임을 증명함.
